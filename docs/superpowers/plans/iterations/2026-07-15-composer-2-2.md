The complete implementation is accepted only when:

- Compact, Expanded, and Scrolling are content-driven states of one Composer Shell rather than separate visual surfaces.

- Every implementation slice updates this handoff and lands in a local commit; no work is pushed.

## Slice 2 — real Tools menu and Prompt Library

Status: implementation and Root verification complete; this handoff accompanies the local Slice 2 commit based on `8ae07a72`.

### Observable result

- The Tools main view now presents three real primary entries in the approved order: Upload image, Upload file, and Prompt Library. Existing visible session actions remain available after the primary group.
- Image and file entries call the existing attachment pipeline with a typed chooser mode. Image accepts PNG/JPEG/WebP/HEIC/HEIF; File retains the existing document/code/data formats, including SVG as a file. The change handler defensively rejects files outside the selected type even if a browser ignores `accept`.
- Image and file disabled states are independent. The existing `appendAttachments` path remains the final authority for the three-image/five-file limits, and the combined Add-more attachment entry still opens the original all-types chooser.
- Prompt Library is a real drill-in over `usePromptStore().search()`, not prototype data. All, My prompts, and Official use the store's existing `isUser` source marker; search and selection reuse the current prompt and chat-command semantics.
- Selecting a prompt closes Tools, replaces the current draft with the trimmed real prompt content, reuses command matching, and restores textarea focus. No prompt data is copied into new state or persistence.
- Escape in Prompt Library returns to Tools and restores focus to the Prompt Library trigger; a second Escape closes Tools and restores focus to Add. Search keeps native Arrow/Home/End editing while Tab remains trapped inside the dialog.
- Tools and Model are mutually exclusive in both directions. The composer is raised above the model backdrop while Model is open so one physical click on Add closes Model and opens Tools; one click on Model closes Tools and opens Model.
- The Tools surface is `268px` on desktop and at most `280px` on compact screens, with `52px` primary rows, localized title/description copy, a `44px` back target, `38px` search field, and a scroll-bounded Prompt list. The nested primary gradient card was removed.

### Changed contracts

- `app/components/chat.tsx`: Tools subview state, typed upload handlers, real Prompt Library, independent capacity state, focus collection/return, nested Escape, and bidirectional Tools/Model switching.
- `app/components/chat.module.scss`: lightweight Tools geometry, primary row copy, Prompt Library controls/list, compact width, bounded height, and composer trigger availability above the model backdrop.
- `app/utils/file.ts`: backwards-compatible `AttachmentUploadKind`, separated accept strings, and defensive post-selection type filtering.
- `app/utils/composer-prompt-library.ts`: deterministic real-prompt source filtering.
- `app/locales/cn.ts` and `app/locales/en.ts`: Upload image/file and Prompt Library titles, descriptions, filters, search, back, unavailable, and empty states.
- `test/composer-tools-menu.test.ts` and `test/gemini-visual-migration.test.ts`: typed chooser, real filters, state wiring, localization, geometry, focus, independent limits, and updated attachment contracts.
