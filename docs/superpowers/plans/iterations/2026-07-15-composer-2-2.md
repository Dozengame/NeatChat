## Slice 1 — unified shell and textarea continuity

Status: implementation and Root verification complete; this handoff accompanies the local Slice 1 commit.

### Observable result

- The outer composer is the single visible Grid Shell across empty/new-chat and existing conversations.
- Compact is exactly `64px` high with Add, textarea, Model, and Send in normal grid flow.
- Real multiline content moves the textarea to the full-width row and places controls in the bottom row without a shape jump.
- Textarea height comes from the browser's real `scrollHeight`; empty drafts use the resolved line height so a wrapped placeholder cannot expand the composer.
- Long drafts stop at `min(174px, 24dvh)` and scroll only inside the textarea.
- Attachments occupy their own normal-flow row; attachment count changes schedule a new textarea measurement.
- Opening Tools or Model freezes the current textarea shape until the menu closes, avoiding menu-induced draft reflow.
- The shell is a semantic `div`; a `label` no longer wraps unrelated buttons.
- Model-menu placement anchors to the whole Composer Shell rather than the former inner label.
- User font sizes keep a proportional `1.5` line height with a `24px` floor, covering the existing `12–40px` setting without clipping large glyphs.
